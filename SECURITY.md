# Security Policy 🛡️

We take the security of Recon, our backend database layers, integrations, and user data extremely seriously. If you believe you have found a security vulnerability in this project, please report it to us privately following the process outlined below.

## Supported Versions

Only the latest version on the `main` branch is actively supported with security updates.

| Version | Supported |
| ------- | --------- |
| >= 0.1.x| ✅ Yes    |
| < 0.1.x | ❌ No     |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security-related bugs. Instead, report any vulnerability via email to **akarshkushwaha.dev@gmail.com**.

When reporting, please include:
- A detailed description of the vulnerability.
- Step-by-step instructions or a Proof of Concept (PoC) to reproduce the issue.
- The potential impact of the vulnerability.

We will acknowledge receipt of your report within 48 hours and work with you to resolve the issue as quickly as possible.

## Thank You!

Thank you to all security researchers and developers who help keep Recon safe, secure, and resilient for teams around the world! 📡
